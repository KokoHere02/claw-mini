# Agent 测试问题清单

## 目的

这份清单用于验证当前 agent 是否真的按多轮 loop 方式工作，而不是只做一次模型调用。

重点观察：

1. 是否会先拿中间结果，再继续下一步
2. 是否能基于工具结果继续推理
3. 是否在工具失败后切换策略或正确收尾
4. 是否能从文件/命令结果中提取最终答案

## 一、单工具基础测试

### 1. 当前时间

```text
现在几点？
```

预期：

- 调用 `get_current_time`
- 返回当前时间

### 2. 表达式计算

```text
计算 `(23 + 19) * 4`
```

预期：

- 调用 `calculate_expression`
- 返回正确结果

### 3. 读取 package.json

```text
读取 "package.json"
```

预期：

- 调用 `run_command`
- 使用 `cat` 或等价文件读取行为
- 返回文件内容摘要或关键内容

### 4. 列出当前目录

```text
列出当前目录
```

预期：

- 调用 `run_command`
- 使用 `ls` 或等价行为
- 返回目录列表

### 5. 抓取网页

```text
请求 https://example.com
```

预期：

- 调用 `http_request`
- 返回网页内容摘要

## 二、多轮 loop 测试

### 6. 先读文件，再回答

```text
先读取 "package.json"，再告诉我项目名和依赖里有没有 ai
```

预期：

- 第一轮先读取 `package.json`
- 后续轮基于读取结果回答项目名和依赖信息
- 不应跳过读取直接猜

### 7. 先列目录，再判断入口

```text
先列出当前目录，再读取你认为最可能是入口的文件，并告诉我程序从哪里启动
```

预期：

- 先列目录
- 再决定读哪个文件
- 最后结合结果回答入口位置

### 8. 总结 runAgent

```text
读取 "src/agent/index.ts"，再总结 runAgent 做了哪几步
```

预期：

- 先读文件
- 再基于文件内容做结构化总结

### 9. 查配置来源

```text
先读取 "src/config.ts"，再告诉我 agent 最大步数是从哪里配置的
```

预期：

- 先读 `src/config.ts`
- 再回答 `config.agent.maxSteps` 和对应环境变量/配置来源

### 10. 条件判断式测试

```text
先读取 "package.json"，如果没有 test 脚本就直接告诉我原因
```

预期：

- 先读取文件
- 再判断 `scripts.test` 是否存在或是否只是占位
- 最终给结论

## 三、失败恢复测试

### 11. 不存在的文件

```text
读取 "not-exists.txt"，如果失败就告诉我失败原因
```

预期：

- 先尝试读取
- 失败后不要胡编文件内容
- 应明确说明失败原因

### 12. 无效域名

```text
请求 https://not-a-real-domain.invalid ，失败的话直接总结限制
```

预期：

- 先尝试请求
- 失败后说明请求失败，而不是编造网页内容

### 13. 失败后切换策略

```text
读取一个不存在的文件，如果失败就改为列出当前目录
```

预期：

- 第一轮尝试读文件
- 失败后进入下一轮
- 第二轮改成列目录
- 最终基于列目录结果回答

## 四、建议优先测试的 5 条

如果你时间有限，优先测这 5 条：

1. `现在几点？`
2. `计算 `(23 + 19) * 4``
3. `先读取 "package.json"，再告诉我项目名和依赖里有没有 ai`
4. `先列出当前目录，再读取你认为最可能是入口的文件，并告诉我程序从哪里启动`
5. `读取一个不存在的文件，如果失败就改为列出当前目录`

## 五、你该怎么看结果

你判断 agent 是否真的有 loop，重点看：

1. 是否有“先取中间结果，再继续”的行为
2. 是否会基于工具结果继续下一轮，而不是一次性猜完
3. 工具失败后是否会调整策略
4. 最终答案是否明显依赖前面工具拿到的数据

## 六、最像真实使用场景的问题

如果你想测得更贴近实际，可以直接问：

```text
先读取 "src/config.ts" 和 "src/agent/index.ts"，再告诉我这个项目里的 agent loop 是怎么工作的
```

这个问题最能看出：

- 它会不会连续调用多个工具
- 会不会综合多个结果
- 会不会在最终回答里正确引用上下文
